# UX Blueprint — SciPlotter

> กติกาการออกแบบ UI/UX ของทั้งโปรเจค — ฟีเจอร์ใหม่ทุกตัวต้องหาที่อยู่ของตัวเอง
> ตามโมเดลนี้ ห้ามซ่อนความสามารถหลักไว้ใน panel ที่มองไม่เห็น

## โมเดลความคิดหลัก: 3 ขั้น (มองเห็นครบทุกขั้นตลอดเวลา)

```
① ข้อมูล            ② เลือกคอลัมน์ → พล็อต        ③ ดู/แต่งกราฟ
   เปิดไฟล์ หรือ         เลือก X, Y แล้วกดปุ่ม           Graph window ใน MDI
   พิมพ์ใน Book1          พล็อตเส้น/จุด                  + เครื่องมือ (zoom/crosshair)
```

- **Panel ซ้าย (Data) = กระดูกสันหลังของ workflow** — การ์ด ①②③ เรียงบน-ล่าง
  ผู้ใช้ใหม่อ่านจากบนลงล่างแล้วใช้แอปเป็นทันที
- **Book1 (worksheet) เป็นจุดเริ่มได้ด้วยตัวเอง** — พิมพ์ข้อมูลลงตาราง แล้วใช้
  แถบเครื่องมือบนชีต (`ใช้ข้อมูลนี้` / `พล็อตเส้น` / `พล็อตจุด`) หรือคลิกขวา
  ไม่ต้องรู้จัก panel อื่นเลยก็พล็อตได้ (แบบ OriginPro)
- **Graph window = ผลลัพธ์** — พล็อตแล้วหน้าต่างกราฟถูกยกขึ้นหน้าเสมอ

## ที่อยู่ของแต่ละอย่าง

| สิ่งที่ผู้ใช้ต้องการ | อยู่ที่ไหน |
|---|---|
| เปิดไฟล์ข้อมูล | ปุ่มใหญ่ การ์ด ① / เมนู File / ลากไฟล์วาง |
| พิมพ์ข้อมูลเอง | Book1 + แถบเครื่องมือบนชีต (เพิ่มแถว/คอลัมน์) |
| ใช้ข้อมูลที่พิมพ์ | ปุ่ม "ใช้ข้อมูลนี้" บนชีต หรือ "ใช้ข้อมูลจากตาราง" การ์ด ① |
| สลับไฟล์ที่โหลดไว้หลายไฟล์ | ลิสต์ในการ์ด ① (เพิ่ม/ใช้/ลบ) |
| เลือกแกน X/Y + พล็อต | การ์ด ② (CompactPlotPanel) — **มองเห็นตลอด ไม่ถูกซ่อน** |
| พล็อตเร็วจากคอลัมน์ที่เลือกในชีต | คลิกขวาบนตาราง หรือปุ่มพล็อตบนแถบชีต (คอลัมน์แรกที่เลือก = X) |
| เครื่องมือกราฟ (crosshair/zoom) | การ์ด ③ |
| การแปลง/วิเคราะห์ข้อมูล | เมนู Process (Cleaning/Filters) และ Analysis |
| จัดการ Layers ของกราฟ | Inspector ด้านขวา (เสริม — เปิดจาก View/toolbar) |
| จัดการหน้าต่าง Book/Graph | Project Explorer (dock ซ้าย) + เมนู Window |

## กติกาสำหรับฟีเจอร์ใหม่

1. **ห้ามซ่อนขั้นตอนหลัก** — อะไรที่อยู่ใน flow เปิดข้อมูล→พล็อต ต้องมองเห็นโดยไม่ต้องเปิด dock/panel เพิ่ม
2. **Inspector คือของเสริม** — ใส่ได้เฉพาะเครื่องมือปรับแต่งขั้นสูง (layers, style)
   ห้ามใส่สิ่งที่จำเป็นต่อ flow หลัก
3. **การแปลงข้อมูล = เมนู Process, การวิเคราะห์ = เมนู Analysis** — เพิ่มเป็น
   submenu ตามหมวด ไม่สร้างปุ่มกระจัดกระจาย
4. **ผลของ action ต้องมองเห็น** — แปลงข้อมูลแล้วต้องสะท้อนใน Book + status bar
   บอกว่าเกิดอะไรและขั้นถัดไปคืออะไร
5. **โมดูลเฉพาะทาง (Gas Sensor ฯลฯ)** — เป็น activity ใหม่ใน activity rail
   (context panel ของตัวเอง) ไม่ยัดเพิ่มในการ์ด ①②③

## สิ่งที่แก้ไปแล้ว (2026-07)

- ย้าย CompactPlotPanel (X/Y + ปุ่มพล็อต) จาก Inspector ที่ซ่อนอยู่ → การ์ด ②
- Book1 มีแถบเครื่องมือ + คลิกขวา: ใช้ข้อมูล/พล็อต/เพิ่มแถว-คอลัมน์ ได้จากชีตตรง ๆ
- `WorkbookWidget.dataframe()` ตัดแถวว่างล้วนทิ้ง → พิมพ์ไม่กี่แถวบนชีต 32 แถวก็พล็อตสะอาด
- Panel ซ้ายเรียงเป็นการ์ด ①②③ + ปุ่มใหญ่ "เปิดไฟล์ข้อมูล…"
