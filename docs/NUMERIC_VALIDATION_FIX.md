# การแก้ไขปัญหาการตรวจสอบข้อมูลตัวเลขใน SciPlotter

## ปัญหาที่พบ
โปรแกรม SciPlotter แสดงข้อความ "คอลัมน์ที่เลือกไม่มีข้อมูลตัวเลขที่ใช้พล็อตได้" ทั้งๆ ที่คอลัมน์มีข้อมูลตัวเลขอยู่ เนื่องจาก:

1. **การตรวจสอบข้อมูลไม่ละเอียด**: โปรแกรมไม่แสดงสาเหตุที่แท้จริงว่าทำไมข้อมูลไม่สามารถใช้ได้
2. **ข้อความแจ้งเตือนไม่ชัดเจน**: ไม่บอกว่าปัญหาอยู่ที่ไหน
3. **ไม่มี debug information**: ไม่มีข้อมูลสำหรับการแก้ไขปัญหา

```python
    # ลองแปลงเป็นตัวเลขและนับจำนวนที่ใช้ได้
    # ส่งกลับผลลัพธ์พร้อมข้อความอธิบาย
```

### 2. ปรับปรุงฟังก์ชัน `_get_xy()`
- เพิ่มการตรวจสอบข้อมูลก่อนประมวลผล
- เพิ่ม debug messages ที่ละเอียดขึ้น
- แสดงข้อมูล dtype และตัวอย่างข้อมูล
- แสดงจำนวนข้อมูลที่ใช้ได้/ไม่ใช้ได้

### 3. ปรับปรุงข้อความแจ้งเตือน
ข้อความแจ้งเตือนใหม่จะบอกว่า:
- คอลัมน์ไหนมีปัญหา
- ปัญหาคืออะไร (ว่าง, ไม่สามารถแปลงเป็นตัวเลขได้, มีข้อมูลน้อยเกินไป)
- เปอร์เซ็นต์ข้อมูลที่ใช้ได้
- คำแนะนำในการแก้ไข

## ตัวอย่างข้อความแจ้งเตือนใหม่

### กรณีข้อมูลดี
```
✅ คอลัมน์ 'temperature' มีข้อมูลตัวเลข 1000/1000 (100.0%)
```

### กรณีข้อมูลมีปัญหา
```
❌ คอลัมน์ 'mixed_data' มีข้อมูลตัวเลขเพียง 500/1000 (50.0%)
```

### กรณีไม่สามารถแปลงเป็นตัวเลขได้
```
❌ คอลัมน์ 'text_data' ไม่สามารถแปลงเป็นตัวเลขได้
```

## การใช้งาน

### 1. Debug Messages
เมื่อเกิดปัญหา โปรแกรมจะแสดง debug messages ใน console:
```
Debug: X column 'time' - dtype: datetime64[ns], sample: ['2023-01-01T00:00:00.000000000' '2023-01-01T00:01:00.000000000' '2023-01-01T00:02:00.000000000']
Debug: Y column 'value' - dtype: object, sample: ['1.5' 'invalid' '3.5']
Debug: Y conversion - original: 1000, NaN after conversion: 500, valid: 500
```

### 2. ข้อความแจ้งเตือนที่ละเอียดขึ้น
แทนที่จะแสดงข้อความทั่วไป:
```
"คอลัมน์ที่เลือกไม่มีข้อมูลตัวเลขที่ใช้พล็อตได้"
```

จะแสดงข้อความที่ละเอียดขึ้น:
```
"ไม่สามารถแปลงข้อมูลเป็นตัวเลขได้:
• X column 'time' (dtype: datetime64[ns])
• Y column 'value' (dtype: object)

ลองใช้ 'กำหนดชนิดคอลัมน์' เพื่อแปลงข้อมูลก่อน"
```

## การทดสอบ

ใช้ไฟล์ `test_numeric_validation.py` เพื่อทดสอบ:

```bash
cd SciPlotter
python test_numeric_validation.py
```

ไฟล์ทดสอบจะสร้างข้อมูลหลากหลายประเภท:
- ข้อมูลตัวเลขดี
- ข้อมูลผสม (ตัวเลข + ข้อความ)
- ตัวเลขในรูปสตริง
- ข้อมูลข้อความ
- คอลัมน์ว่าง
- ข้อมูลบางส่วนเป็นตัวเลข
- ข้อมูล boolean

## ข้อดีของการแก้ไข

1. **ข้อความแจ้งเตือนชัดเจน**: ผู้ใช้ทราบว่าปัญหาอยู่ที่ไหน
2. **Debug information**: มีข้อมูลสำหรับการแก้ไขปัญหา
3. **การตรวจสอบละเอียด**: ตรวจสอบข้อมูลก่อนประมวลผล
4. **คำแนะนำการแก้ไข**: แนะนำวิธีแก้ไขปัญหา
5. **รองรับข้อมูลหลากหลายประเภท**: จัดการข้อมูลได้ดีขึ้น

## วิธีแก้ไขปัญหาที่พบบ่อย

### 1. ข้อมูลเป็นสตริง
**ปัญหา**: คอลัมน์มีตัวเลขแต่เป็นสตริง (เช่น "1.5", "2.0")
**วิธีแก้**: ใช้ฟีเจอร์ "กำหนดชนิดคอลัมน์" เพื่อแปลงเป็น Float

### 2. ข้อมูลผสม
**ปัญหา**: คอลัมน์มีทั้งตัวเลขและข้อความ
**วิธีแก้**: ทำความสะอาดข้อมูลก่อน หรือเลือกเฉพาะส่วนที่เป็นตัวเลข

### 3. ข้อมูลมี NaN เยอะ
**ปัญหา**: คอลัมน์มีค่า NaN มากกว่า 50%
**วิธีแก้**: ตรวจสอบแหล่งข้อมูลหรือกรองข้อมูลที่ใช้ได้

### 4. คอลัมน์ว่าง
**ปัญหา**: คอลัมน์ไม่มีข้อมูลเลย
**วิธีแก้**: ตรวจสอบการโหลดข้อมูลหรือเลือกคอลัมน์อื่น

## การใช้งานในอนาคต

การแก้ไขนี้จะช่วยให้:
- ผู้ใช้เข้าใจปัญหาได้ง่ายขึ้น
- แก้ไขปัญหาได้เร็วขึ้น
- ลดความสับสนในการใช้งาน
- เพิ่มประสิทธิภาพการทำงานกับข้อมูล
