# การแก้ไขปัญหาการอ่านไฟล์ขนาดใหญ่ใน SciPlotter

## ปัญหาที่พบ
โปรแกรม SciPlotter ไม่สามารถอ่านไฟล์ CSV ที่มีข้อมูลเยอะมากๆ ได้ เนื่องจาก:

1. **ไม่มี chunking mechanism**: โปรแกรมอ่านไฟล์ทั้งหมดเข้า memory ทันที
2. **ไม่มี memory limits**: ไม่มีการจำกัดขนาดข้อมูลที่อ่านได้
3. **ไม่มี progress indicator**: ไม่มีการแสดงความคืบหน้าเมื่ออ่านไฟล์ใหญ่

## การแก้ไขที่ทำ

### 1. แก้ไข `file_io.py`
- เพิ่มฟังก์ชัน `_read_csv_chunked()` สำหรับอ่านไฟล์ขนาดใหญ่แบบ chunking
- แก้ไขฟังก์ชัน `read_csv()` ให้ตรวจสอบขนาดไฟล์ก่อนอ่าน
- เพิ่มการจำกัดจำนวนแถวสูงสุด (1 ล้านแถว) เพื่อป้องกัน memory overflow
- เพิ่มการแสดงความคืบหน้าในการอ่านไฟล์

### 2. แก้ไข `loaders.py`
- เพิ่มฟังก์ชัน `_read_csv_chunked_simple()` สำหรับอ่านไฟล์ขนาดใหญ่
- แก้ไขฟังก์ชัน `load_tabular()` ให้รองรับไฟล์ขนาดใหญ่
- เพิ่มการแสดงข้อมูลขนาดไฟล์ในผลลัพธ์

### 3. แก้ไข `main.py`
- เพิ่มการแสดงข้อมูลขนาดไฟล์และจำนวนแถวใน UI
- ปรับปรุงข้อความสถานะให้แสดงข้อมูลที่ละเอียดขึ้น
- แก้ไข linter errors

## คุณสมบัติใหม่

### การจัดการไฟล์ขนาดใหญ่
- **ไฟล์ขนาดใหญ่**: ไฟล์ที่ใหญ่กว่า 100 MB จะถูกอ่านแบบ chunking
- **Chunk size**: อ่านทีละ 10,000 แถว
- **Memory limit**: จำกัดสูงสุดที่ 1 ล้านแถว
- **Progress indicator**: แสดงความคืบหน้าในการอ่านไฟล์

### การแสดงข้อมูลใน UI
- แสดงขนาดไฟล์ (MB)
- แสดงจำนวนแถวและคอลัมน์
- แสดงสถานะการโหลดที่ละเอียดขึ้น

## การใช้งาน

### สำหรับไฟล์ขนาดเล็ก (< 100 MB)
โปรแกรมจะอ่านไฟล์แบบปกติเหมือนเดิม

### สำหรับไฟล์ขนาดใหญ่ (≥ 100 MB)
โปรแกรมจะ:
1. แสดงข้อความ "ไฟล์ขนาดใหญ่ (X.X MB) - กำลังอ่านแบบ chunking..."
2. อ่านไฟล์ทีละ chunk พร้อมแสดงความคืบหน้า
3. แสดงข้อความ "อ่านแล้ว X,XXX แถว..." ทุกครั้งที่อ่าน chunk ใหม่
4. แสดงข้อความ "อ่านไฟล์เสร็จสิ้น: X,XXX แถว, X คอลัมน์" เมื่อเสร็จสิ้น

## การทดสอบ

ใช้ไฟล์ `test_large_file.py` เพื่อทดสอบการทำงาน:

```bash
cd SciPlotter
python test_large_file.py
```

ไฟล์ทดสอบจะสร้างไฟล์ CSV ขนาดเล็กและใหญ่เพื่อทดสอบการทำงานของฟังก์ชันที่แก้ไขแล้ว

## ข้อดีของการแก้ไข

1. **รองรับไฟล์ขนาดใหญ่**: สามารถอ่านไฟล์ที่มีข้อมูลเยอะมากๆ ได้
2. **ป้องกัน memory overflow**: มีการจำกัดจำนวนแถวสูงสุด
3. **แสดงความคืบหน้า**: ผู้ใช้ทราบสถานะการโหลดข้อมูล
4. **ข้อมูลละเอียด**: แสดงขนาดไฟล์และจำนวนแถวใน UI
5. **Backward compatible**: ไฟล์ขนาดเล็กยังทำงานเหมือนเดิม

## ข้อจำกัด

- ไฟล์ที่มีมากกว่า 1 ล้านแถวจะถูกตัดทอน
- การอ่านไฟล์ขนาดใหญ่อาจใช้เวลานานขึ้น
- ต้องมีหน่วยความจำเพียงพอสำหรับการประมวลผลข้อมูล
