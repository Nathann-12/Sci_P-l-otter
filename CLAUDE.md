# CLAUDE.md — คู่มือสำหรับ AI ที่ทำงานกับโปรเจคนี้

> อ่านไฟล์นี้ก่อนเริ่มงานทุกครั้ง แล้วทำตามกติกาในนี้อย่างเคร่งครัด

## โปรเจคนี้คืออะไร
**SciPlotter** = เครื่องมือเดสก์ท็อปวิเคราะห์/พล็อตข้อมูลวิทยาศาสตร์ (PySide6 + Matplotlib + pandas/numpy/scipy)
เป้าหมายระยะยาว: เป็น **Research OS สำหรับนักวิจัยไทย** — เปิดข้อมูล → วิเคราะห์ → ทำกราฟ → โมดูลเฉพาะทาง → เขียนรายงาน → reproduce ได้
ขอบเขตฟีเจอร์ทั้งหมด + สถานะปัจจุบัน อยู่ใน **[ROADMAP.md](ROADMAP.md)** (แหล่งความจริงเดียว)

## วิธีรันและทดสอบ (สำคัญ — ใช้ interpreter ตัวนี้เท่านั้น)
- venv ของโปรเจคคือ `.venv/Scripts/python.exe` (Python 3.11.9) — **อย่าใช้ `py`/`python` ระบบ** (เคยพังเพราะ venv เก่าชี้ interpreter ที่ถูกลบ; ดู `memory/env-and-run.md` ฝั่งผู้ใช้)
- **รันเทสต์ (headless):**
  ```bash
  QT_QPA_PLATFORM=offscreen .venv/Scripts/python.exe -m pytest tests/ -q
  ```
  Baseline ปัจจุบัน (ตรวจล่าสุด 2026-07-18): **1281 passed, 3 skipped**
- **เปิดแอป (GUI):** `.venv/Scripts/pythonw.exe main.py` (หรือ `python main.py`)
- **smoke import:** `QT_QPA_PLATFORM=offscreen .venv/Scripts/python.exe -c "import main; main.MainWindow"`
- ถ้า dependency หาย: `.venv/Scripts/python.exe -m pip install -r requirements.txt`

**ชั้น UI (โมเดล OriginPro เต็มรูปแบบ — สำคัญ)**
- **UX ทั้งแอปยึด loop แบบ Origin** (ดู [docs/UX_FLOW.md](docs/UX_FLOW.md) — บังคับสำหรับฟีเจอร์ใหม่): เปิดไฟล์/พิมพ์ใน Book → เลือกคอลัมน์บนชีต → คลิกไอคอนพล็อตใน function bar สองแถวด้านบนหรือปุ่มบนชีต → **Graph window ใหม่เสมอ**; ไม่มีแผง workflow ซ้ายแบบถาวร — Worksheet + two-row function toolbar + Project Explorer แบบ parked side tab + เมนู เท่านั้น; **ห้ามซ่อนความสามารถหลักไว้ใน dock/panel ที่ปิดอยู่**
- **Multi-book แบบ Origin**: 1 ไฟล์ = 1 Book (เปิด/ลากไฟล์ → `_stage_insert` → `_open_book_for_dataset`); คลิกหน้าต่าง Book = สลับข้อมูล (`MdiWorkspace.bookActivated` → `_on_book_activated` ตั้ง `self.workbook`/`self._df`); DataFrame ของแต่ละ Book อยู่ที่ `WorkbookWidget.source_df` + registry `_datasets` (session ใช้ตัวนี้)
- **workspace กลาง = MDI** ([UI/mdi_workspace.py](UI/mdi_workspace.py) `MdiWorkspace`) — Book/Graph เป็น QMdiSubWindow ลอยได้แบบ Origin. `MdiWorkspace` **เลียนแบบ API ของ TabManager** และถูกตั้งเป็น `self.tabs` → โค้ดพล็อต (plot/plotcore/view mixin) reuse ได้โดยไม่ต้องแก้. MainWindow เปิดแบบ **sheet-first** (`start_with_graph=False`): มี Book1 อย่างเดียว, กราฟใหม่ = `self.tabs.add_tab()` ตอนผู้ใช้สั่ง plot; worksheet = `add_book()`. **อย่าเขียนโค้ดพล็อตให้ผูกกับ QTabWidget** — ใช้ API ของ `self.tabs`
- **worksheet แบบ Excel** ([widgets/workbook.py](widgets/workbook.py) `WorkbookWidget`) — แถว Long Name/Units/Comments/F(x)=; **หัวคอลัมน์ถือ designation แบบ Origin** (`A(X)`/`B(Y)`/`C`=Disregard, คลิกขวาหัวคอลัมน์ → Set As, โหลดไฟล์แล้วคอลัมน์เวลาเป็น X อัตโนมัติ); แถบเครื่องมือ/คลิกขวาบนชีต emit `use_data_requested`/`plot_requested`/`overlay_requested` → `adopt_workbook_data` (data mixin) / `plot_from_workbook(style, new_graph)` (plot mixin — เส้นทางพล็อตจากชีตทั้งหมดรวมที่นี่)

Top-level ของ Process ห้ามมีกอง action ตรง ๆ; ทุกคำสั่งต้องมี icon/status tip เท่าที่ทำได้และมี QAction user-flow tests ผ่าน path หมวดจริง

Mapping ไอคอนอยู่ที่ `_QTA_ICON_MAP` ใน `main.py` (qtawesome MDI)

- **Graph-scoped toolbar contract** — คำสั่ง toolbar ที่แตะกราฟ (`Plot`, `Spectrogram`, `Export Figure/Data`, `Crosshair`, `Box Zoom`, `Format Graph`, `Plot from Equation`, `Processors/FFT`) ต้อง resolve ไปที่ Graph window ที่เลือกอยู่ หรือ Graph ล่าสุดที่ผู้ใช้เลือกเมื่อ Book โฟกัสอยู่; ห้าม fallback ไป Graph 1 ตามตำแหน่งโดยไม่ตั้งใจ
- **Action enablement contract** (`main_window_toolbar_mixin.py::update_action_states` + `_ACTIONS_NEED_DATA`/`_ACTIONS_NEED_GRAPH`) — แบบ Origin: toolbar command ที่ยังใช้ไม่ได้ต้อง **dim (disabled)** ไม่ใช่กดแล้วเด้ง dialog ปฏิเสธ. `_ACTIONS_NEED_DATA` = enabled ต่อเมื่อ active Book มีข้อมูล (`_resolve_active_dataframe()` ไม่ว่าง); `_ACTIONS_NEED_GRAPH` = enabled ต่อเมื่อมี Graph window (`len(tabs.tabs)>0`); ที่เหลือ always-on (open/batch_import/settings/add_row/addtab/plot_equation/workflow_*). Key ที่อยู่ทั้งสอง set แชร์ QAction object เดียว → dim พร้อมกันทุก surface (top bar + docks). Disabled action **`trigger()` ไม่ยิง** → user-flow tests ต้อง seed data/สร้าง graph ก่อน trigger (ที่มีอยู่ทำอยู่แล้ว). Refresh ผ่าน `_refresh_action_states()` ที่ท้าย `load_columns_from_df`/`_on_book_activated` + signal `tabCreated`/`tabRemoved`/`currentChanged`/`bookActivated` (wired ใน `_wire_action_state_updates`); เพิ่ม action/feature ใหม่ต้องจัดหมวดให้ถูก (ไม่งั้นมันจะ enabled ตลอดหรือ dim ตลอดผิด ๆ). Data command ที่ต้องมีกราฟ (FFT/PSD/spectrogram) ใช้ `_ensure_graph_canvas()` เพื่อ **สร้าง Graph ให้เองเมื่อมีข้อมูลแต่ยังไม่มีกราฟ** ไม่ปฏิเสธ
- **Plot Details decoration contract** (`dialogs/plot_details_dialog.py` + `core/plot_style.py`) — live Apply/Preset ห้ามเปลี่ยน live figure size/DPI; print size/DPI เก็บใน `GraphTab._print_figure` เพื่อ export เท่านั้น. หลัง apply ต้อง **relayout ด้วย `tight_layout` เท่านั้น ห้าม `set_size_inches` บน live canvas** — การ force ขนาดจาก Qt *logical* pixels ทำให้ figure เล็กกว่า widget เท่า devicePixelRatio บนจอ HiDPI (Windows 125%/150%) = บั๊ก "กราฟย่อเวลาตกแต่ง" (`_relayout_live_figure` ใน `main_window_plotstyle_mixin.py`). Decoration engine ครอบคลุมระดับ Origin: title/label typography (color/bold/italic/family/pad), ticks (length/width/color/label-color/rotation/minor/mirror/sci-notation), tick-label display override (`tick_labels`: decimal/scientific/engineering/percent, decimal places, divide factor, **formula เช่น `2*x` ผ่าน `compile_tick_formula` — AST whitelist เท่านั้น ห้ามแทนด้วย eval ตรง และ formula ชนะ divide factor**, prefix/suffix, plus/minus, thousands), แท็บ Scale แบบ Origin (From/To/Type/Reverse ต่อแกน, rescale margin % ตอน autoscale, Major "By Increment" + **Anchor Tick** ผ่าน `MultipleLocator(offset)`, Minor "By Increment" หรือ "By Counts" — increment ชนะ counts), per-side spine visibility, reference lines (gid `_ps_refline_*` — re-apply ต้องไม่ stack, มี label/width/opacity), grid major+minor แยก color/style/width/alpha + axis, legend title/spacing/markerscale/pads, per-line marker face/edge/width + fillstyle/drawstyle/zorder + glow/shadow. **Scientific palettes + complete journal presets:** `SCIENTIFIC_PALETTES` (8 curated, 5 colorblind-safe) + `apply_palette(ax, name, line_width=)` recolour ทุกเส้นตามลำดับ (tint marker + rebuild legend); `JOURNAL_PRESETS` เป็น publication style ครบ (open spines/tick dir/minor/family/grid off/CB-safe palette+line width) ไม่ใช่แค่ font size — preset พก `palette`/`line_width` ที่ `_on_apply_preset` pop ออกไปตั้ง palette combo. **palette เป็น one-shot action ไม่อยู่ใน `read_style`/`diff_style`** — mixin apply เมื่อ `style["palette"] != baseline["palette"]` เท่านั้น (default `"— keep colors —"` = no-op); AI `format_graph` รับ `journal_preset/palette/colorblind/line_width` เรียก apply_style+apply_palette ตรง. **Live preview:** dialog ต่อทุก control (`_wire_live_preview` — QLineEdit/spin/checkbox/ColorButton + palette combo; **ยกเว้น** cb_line/cb_preset/cb_template ที่แค่ repopulate) เข้า debounce `QTimer` 180ms → emit `applied` → apply เอง ไม่ต้องกดปุ่ม (มี checkbox "Live preview" default on). programmatic load (preset/template/สลับ curve) ต้อง set `dlg._loading=True` กัน feedback loop; `finished`→stop timer กัน apply หลังปิด. **Cancel ต้องย้อน** — `open_plot_details_dialog` เก็บ snapshot ตอนเปิด แล้ว `_restore_plot_details` (apply_style ทั้งก้อน + per-line) เมื่อ reject เพราะ live commit ไปแล้ว. **Decoration suite:** per-line `fill` (under/between_next) + `value_labels` (auto-thin ≤200) + `errorbar_mode` (constant/percent) เป็น gid-tagged artists (`_ps_fill_*`/`_ps_vlab_*`/`_ps_err_*`) — re-apply ต้อง replace ไม่ stack, ปิด = ลบ; axes-level section `inset` (zoom panel + `indicate_inset_zoom` — mpl ≥3.10 คืน `InsetIndicator` เดียว ลบทั้งตัว) และ `colorbar` (colormap จาก `COLORMAPS` + สร้าง/ลบ/แต่ง colorbar) อยู่ใน `_ATOMIC_SECTIONS`; 5. **refactor = pure move** — ห้ามแก้พฤติกรรมเงียบ ๆ ระหว่างย้ายโค้ด
6. เทสต์ปัจจุบันส่วนใหญ่เป็น **structure test** (เช็คการต่อสาย) — ฟีเจอร์ใหม่ต้องมี **behavioral test** (เช็คผลลัพธ์จริง) ด้วย
7. commit ทีละหน่วยงานที่มีความหมาย; **อย่า push** เองเว้นแต่ผู้ใช้สั่งชัดเจน (push เข้า `main` ถูก guard ไว้)
8. **ทุกฟีเจอร์/คำสั่งใหม่ต้องเปิดให้ AI เรียกได้ด้วย** — ลงทะเบียนเป็น AI tool ใน [`ai/app_tools.py`](ai/app_tools.py) (ผ่าน `registry.add(...)`) พร้อม behavioral test ใน `tests/test_ai_assistant.py`. handler ต้อง **defensive + ไม่เปิด modal dialog** (รับ argument ตรง ๆ, คืน string สั้นให้โมเดลอ่าน, error กลายเป็นข้อความไม่ throw). ฟีเจอร์ที่มี logic บริสุทธิ์อยู่แล้วให้ทำ adapter แบบ non-interactive เรียก logic นั้นโดยตรง — ห้ามปล่อยฟีเจอร์ใหม่ที่ AI เข้าไม่ถึง

## สูตรเพิ่มฟีเจอร์ (ทำตามแพทเทิร์นเดิม)
ฟีเจอร์ใหม่ปกติ = 6 ชิ้น:
1. **logic** — ฟังก์ชันคำนวณบริสุทธิ์ใน `processors.py`/`analysis/`/`core/` (unit-test ได้)
2. **dialog** (ถ้ามี UI input) ใน `dialogs/`
3. **method ใน mixin** ที่เหมาะ (หรือสร้าง mixin ใหม่ถ้าเป็นโดเมนใหม่ เช่นโมดูลเฉพาะทาง) — ทำหน้าที่ต่อ UI เข้ากับ logic
4. **wire action** ในเมนู/toolbar (`main_window_menu_mixin` / `main_window_toolbar_mixin`)
5. **AI tool** — ลงทะเบียนใน `ai/app_tools.py` ให้ AI เรียกความสามารถนี้ได้ (กติกาข้อ 8)
6. **test** ใน `tests/` (รวม AI tool test) + ติ๊ก ROADMAP

> โมดูลเฉพาะทางใหม่ (Gas Sensor, Electrochemistry, ฯลฯ) ควรเป็น mixin/แพ็กเกจของตัวเอง + logic แยกเป็นโมดูลบริสุทธิ์เพื่อ test ง่าย
