# การแก้ไขปัญหาข้อมูลเวลา (DateTime) ใน SciPlotter

## ปัญหาที่พบ
จากข้อความแจ้งเตือน "คอลัมน์ 'epoch' ไม่สามารถแปลงเป็นตัวเลขได้" แสดงให้เห็นว่าโปรแกรมไม่สามารถจัดการข้อมูลเวลาได้อย่างเหมาะสม เนื่องจาก:

1. **ไม่รู้จักข้อมูลเวลา**: โปรแกรมไม่สามารถจดจำข้อมูล datetime ได้
2. **ข้อความแจ้งเตือนไม่ชัดเจน**: ไม่บอกว่าข้อมูลเวลาใช้ได้สำหรับแกน X
3. **ไม่มีคำแนะนำการแก้ไข**: ไม่แนะนำวิธีจัดการข้อมูลเวลา

## การแก้ไขที่ทำ

### 1. ปรับปรุงฟังก์ชัน `_check_column_numeric()`
เพิ่มการตรวจสอบข้อมูลเวลา:

```python
# ตรวจสอบว่าเป็น datetime หรือไม่
if pd.api.types.is_datetime64_any_dtype(col_data):
    return True, f"คอลัมน์ '{col_name}' เป็นข้อมูลเวลา (datetime) - ใช้ได้สำหรับแกน X"

# ลองแปลงเป็น datetime ถ้าไม่สามารถแปลงเป็นตัวเลขได้
try:
    datetime_data = pd.to_datetime(col_data, errors="coerce")
    datetime_valid_count = datetime_data.notna().sum()
    if datetime_valid_count > 0:
        return True, f"คอลัมน์ '{col_name}' เป็นข้อมูลเวลา (datetime) - ใช้ได้สำหรับแกน X"
except Exception:
    pass
```

## ข้อดีของการแก้ไข

1. **รองรับข้อมูลเวลา**: สามารถจดจำและจัดการข้อมูล datetime ได้
2. **ข้อความแจ้งเตือนชัดเจน**: บอกว่าข้อมูลเวลาใช้ได้สำหรับแกน X
3. **คำแนะนำการแก้ไข**: แนะนำวิธีจัดการข้อมูลที่ไม่เหมาะสม
4. **Debug information**: แสดงข้อมูลสำหรับการแก้ไขปัญหา
5. **รองรับรูปแบบเวลาหลากหลาย**: จัดการข้อมูลเวลาในรูปแบบต่างๆ ได้

## วิธีแก้ไขปัญหาที่พบบ่อย

### 1. ข้อมูลเวลาเป็นสตริง
**ปัญหา**: คอลัมน์มีข้อมูลเวลาแต่เป็นสตริง (เช่น "2023-01-01 12:00:00")
**วิธีแก้**: โปรแกรมจะแปลงอัตโนมัติ หรือใช้ "กำหนดชนิดคอลัมน์" เพื่อแปลงเป็น Datetime

### 2. ข้อมูลเวลาเป็น Unix timestamp
**ปัญหา**: คอลัมน์มีข้อมูลเวลาแต่เป็นตัวเลข (เช่น 1672531200)
**วิธีแก้**: โปรแกรมจะแปลงอัตโนมัติ หรือใช้ "กำหนดชนิดคอลัมน์" เพื่อแปลงเป็น Datetime

### 3. ข้อมูลเวลาไม่ถูกต้อง
**ปัญหา**: คอลัมน์มีข้อมูลที่ไม่ใช่เวลา
**วิธีแก้**: ใช้ "กำหนดชนิดคอลัมน์" เพื่อแปลงเป็น Datetime หรือเลือกคอลัมน์อื่น

### 4. ข้อมูลเวลาและตัวเลขผสม
**ปัญหา**: คอลัมน์มีทั้งข้อมูลเวลาและตัวเลข
**วิธีแก้**: แยกข้อมูลหรือใช้ "กำหนดชนิดคอลัมน์" เพื่อแปลงเป็น Datetime

## การใช้งานในอนาคต

การแก้ไขนี้จะช่วยให้:
- ผู้ใช้เข้าใจว่าข้อมูลเวลาใช้ได้สำหรับแกน X
- แก้ไขปัญหาได้เร็วขึ้น
- ลดความสับสนในการใช้งาน
- เพิ่มประสิทธิภาพการทำงานกับข้อมูลเวลา
- รองรับข้อมูลเวลาหลากหลายรูปแบบ
